Introdução

Este guia contém os principais comandos Git utilizados no dia a dia de desenvolvimento, incluindo exemplos, explicações e boas práticas. Um índice clicável está disponível abaixo para facilitar a navegação.

Configuração Inicial

git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"

Define seu nome de usuário e e-mail para commits

Inicialização

git init

Cria um novo repositório Git no diretório atual.

Status

git status

Verifica o status da branch e exibe os arquivos que sofreram alterações e que já foram colocados na staging area.

Clonar Repositório

git clone https://github.com/usuario/repositorio.git

Faz uma cópia completa de um repositório remoto (como o do GitHub) no seu computador.

Branches

As branches permitem trabalhar em diferentes linhas de desenvolvimento dentro de um mesmo projeto Git.

Criar nova branch

git checkout -b nome-da-branch

Cria e muda automaticamente para uma nova branch chamada nome-da-branch.

Listar branches

git branch

Exibe todas as branches existentes no repositório. A branch atual é destacada com um asterisco (*).

Mudar de branch

git checkout nome-da-branch

Alterna para a branch especificada.

Deletar branch

git branch -d nome-da-branch

Remove a branch local após garantir que ela foi mesclada.

Staging e Commit

A área de staging é onde você prepara as alterações antes de criar um commit. O commit registra essas alterações de forma permanente no histórico do Git.

Adicionar arquivos

git add nome-do-arquivo
git add . # todos os arquivos modificados

Adiciona arquivos específicos (ou todos) à área de staging para serem incluídos no próximo commit.

Criar novo commit

git commit -m "mensagem do commit"

Cria um ponto de salvamento no histórico do projeto com uma mensagem descritiva sobre as alterações realizadas.

Trabalhando com Repositório Remoto GitHub

Esses comandos permitem conectar seu repositório local com um servidor remoto (como o GitHub), enviar e receber alterações.

Verificar remotos

git remote -v

Lista os repositórios remotos configurados e suas URLs.

Enviar alterações

git push origin nome-da-branch

Envia os commits da branch atual para o repositório remoto.

Puxar alterações

git pull origin nome-da-branch

Baixa e aplica as alterações mais recentes do repositório remoto na sua branch local.

Merge

git checkout main #se essa for a branch que deseja fazer o marge
git merge nome-da-branch

Combina as alterações de uma branch específica com a branch atual (geralmente a main).

Limpeza

git checkout -- nome-do-arquivo

[ATENÇÃO] Esse comando restaura a branch para estado do último commit, em outras palavras apaga todas as alterações feitas.

Extras

Comandos adicionais úteis para análise e inspeção de histórico de commits e diferenças entre arquivos.

Histórico de commits

git log

Exibe o histórico completo de commits, com autor, data e mensagens de cada alteração.

Diferença entre arquivos

git diff

Mostra as diferenças entre as versões modificadas e as versões salvas no último commit.

Boas Práticas: Branches

Adotar convenções claras para nomes de branches facilita a organização e o entendimento do propósito de cada uma:

Tipo Prefixo Exemplo
Funcionalidadefeature/feature/login-screen
Correçãofix/fix/empty-password-bug
Refatoraçãorefactor/refactor/password-generator
Documentaçãodocs/docs/update-readme
Testestest/test/unit-user-service
Infraestruturachore/chore/update-dependencies

Dica: use nomes descritivos, em minúsculas e separados por hífens.

Boas Práticas: Commits

Mensagens de commit devem ser curtas, objetivas e usar o tempo verbal imperativo. Padronize com prefixos claros:

Tipo Prefixo Exemplo
Novo recursofeat:feat: add login functionality
Correçãofix:fix: correct input validation on form
Refatoraçãorefactor:refactor: simplify user session handling
Documentaçãodocs:docs: add API usage examples to README
Testestest:test: create unit tests for password module
Chorechore:chore: update dependencies and cleanup

Exemplo completo:

git commit -m "feat: implement user authentication via email"

Comandos para Fazer Merge com a Main

git checkout main
                    git pull origin main
                    git checkout nome-da-branch
                    git merge main

Esse processo garante que sua branch esteja atualizada antes do merge, minimizando conflitos.
